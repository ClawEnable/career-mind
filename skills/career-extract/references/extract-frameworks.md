# Extract Frameworks

These frameworks are used ONLY in Phase 4 (Structuring) to organize collected information. They are NOT questioning scripts. Do not mention these frameworks to the user during conversation.

## SOAR Framework (preferred)

| Element | Question answered | Example |
|---------|------------------|---------|
| **Situation** | What was the context? | "Team needed an automated UI alert system" |
| **Obstacle** | What was the challenge? | "Multi-page concurrent alerts causing display uncertainty" |
| **Action** | What did you specifically do? | "Designed scheduling strategy based on OC Runtime Hook + event-driven model" |
| **Result** | What was the outcome? | "Canary deployment alert accuracy significantly improved" |

## STAR Framework (alternative)

| Element | Question answered |
|---------|------------------|
| **Situation** | Context and background |
| **Task** | What needed to be done |
| **Action** | What you specifically did |
| **Result** | Measurable outcome |

## Usage Rules

1. Use SOAR when the experience has a clear challenge/obstacle
2. Use STAR when the experience is more task-oriented
3. Do NOT ask users to fill in these frameworks. Use them internally to organize what was said
4. Each material entry should naturally cover all four elements after structuring
5. If structuring reveals a missing element → go back to conversation (Phase 3) to fill the gap
