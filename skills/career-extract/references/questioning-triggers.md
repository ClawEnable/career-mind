# Questioning Triggers for Extract

These are follow-up trigger patterns — not a script. When the user's response matches a pattern on the left, use the corresponding follow-up strategy on the right.

## Trigger Patterns

| User response contains | Follow-up strategy |
|-----------------------|-------------------|
| "responsible for" / "participated in" / "worked on" | "What specifically did you do on this?" |
| "good results" / "improved" / "enhanced" | "Do you have specific numbers? For example, from what to what?" |
| "we" / "the team" / "everyone" | "What was your personal contribution to this project?" |
| "nothing special" / "just normal" / "it was okay" | Switch angle: result-reverse → "What did this project ultimately deliver?" or difficulty-reverse → "What was the hardest part?" |
| "and then" / "after that" / "next" | User is narrating chronologically — let them continue, don't interrupt |
| "I think" / "I believe" / "in my opinion" | Capture this as user_verbatim — this is natural expression, valuable for interview prep |
| Specific number or metric | Confirm and ask for context: "How was this metric measured?" |
| Technical term or tool | Ask about the decision: "Why did you choose this approach/tech? Did you consider alternatives?" |
| Comparison ("better than before" / "compared to") | Dig into the baseline: "What was it like before? How big was the change?" |

## Angle-Switch Strategies

When direct questioning hits a wall, switch to one of these angles:

1. **Result-reverse:** "What did this project ultimately produce? What was the business impact?"
2. **Difficulty-reverse:** "What was the hardest part of this project?"
3. **Contrast-reverse:** "What would have happened if this project wasn't done?"
4. **Decision-reverse:** "Were there other approaches considered at the time? Why was this one chosen?"
5. **Stakeholder-reverse:** "Who/which teams did this project affect? What was their feedback?"

## Principles

- One question at a time
- Follow the user's lead, don't force a structure
- These triggers supplement natural conversation, not replace it
- If the user is freely sharing details, don't apply triggers — just listen and capture
