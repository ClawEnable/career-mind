# Extract Frameworks

Internal organizing tools for structuring captured information. Never mention these framework names to the user.

## Three Dimensions

Every entry should cover:

1. **What** — Specific actions taken. Not responsibilities, but concrete work done.
2. **How** — Methods, decisions, approaches. The reasoning behind choices.
3. **Result** — Outcomes, impact, data. What changed because of this work.

Missing dimensions → natural follow-up questions during capture.

## Entry Type Selection

| If the information is... | Use type |
|--------------------------|----------|
| Tied to a company/project with a time range | project |
| A standalone capability or skill demonstration | signal |
| A specific achievement, award, or milestone | achievement |

## Tag Extraction

From user's description, extract:
- **Technologies** mentioned (languages, frameworks, tools)
- **Skills demonstrated** (design, debugging, leadership, communication)
- **Domains** involved (mobile, backend, infrastructure, data)

Tags should be lowercase, hyphenated. Example: ["swift", "performance-optimization", "mobile"]
