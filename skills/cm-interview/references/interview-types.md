# Interview Type Adaptations

Interview prep strategy adapts based on user's role type. Detect automatically from profile.md target_direction or from resume content.

## Adaptation Matrix

| User Type | Deep-dive Focus | Story Prep Priority | Weakness Prep |
|-----------|----------------|--------------------|--------------|
| Technical | Technical decisions, architecture tradeoffs, debugging methodology | System design walkthroughs, specific tech choices | Short tenure explanation, tech stack breadth vs depth |
| Management | Strategic impact, team building, stakeholder management | Organizational change narratives, resource decisions | IC-to-manager transition, scope of impact |
| New Grad | Academic projects, internships, learning ability | Problem-solving approach, teamwork examples | Limited experience, career direction clarity |
| Career Change | Transferable skills, motivation for change, fresh perspective | Cross-domain skill mapping | Employment gaps, why leaving current field |
| Creative | Portfolio decisions, creative process, client outcomes | Project storytelling, design/creative rationale | Quantifying creative impact, business alignment |
| General | Experience depth, self-awareness, growth trajectory | STAR-format behavioral stories | Any gaps or inconsistencies in narrative |

## Universal Elements (all types)

These apply regardless of user type:
- Self-introduction: 30-second and 2-minute versions
- "Tell me about yourself" response framework
- Questions to ask the interviewer (3-5 based on target direction)
- Salary discussion preparation (if target direction is clear)

## Tone Principle

Interview prep uses **user_verbatim** from entries (not polished resume text). Speaking naturally in an interview is more authentic than reciting optimized phrases. If no entries exist, use resume content but encourage the user to rephrase in their own words.
